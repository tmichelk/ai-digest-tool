最新AIニュースをチェックし、価値ある情報があればLINEに送信してください。

**送信の判断基準（以下のいずれかに該当する場合のみ送信する）:**
- 重大な新モデルの発表（GPT・Claude・Geminiなどの新バージョン含む）
- 各AIツールの新機能・アップデートの発表
- 便利な使い方・実践的なTipsの紹介
- 公式が発表した最新情報（ブログ・プレスリリース・Changelog）

**上記に該当する記事が1件も見つからなかった場合は、LINEに何も送らずに処理を終了してください。**

## 参照スキル

以下のスキルを参照してください：
- **rss-fetcher**: RSSフィードの取得方法と出力形式
- **news-analyzer**: 記事の選別・要約・LINEメッセージのフォーマット
- **line-sender**: LINEへの送信方法

## 処理フロー

### 1. 設定ファイルを確認する

`configs/digest.yml` を読み込み、フィードリストと設定を把握する。

### 2. RSSフィードを取得する

**⚠️ 必ずこのスクリプトを使用すること**

```bash
node .claude/skills/rss-fetcher/scripts/fetch-feeds.js
```

出力: `/tmp/raw-feeds.json`

取得失敗のフィードがあっても処理を続行する。

### 3. 記事を選別・要約する

`/tmp/raw-feeds.json` を読み込み、**news-analyzer スキル**の方針に従って：

1. 各カテゴリから最大3件（Cursorは全件）を選ぶ
2. 記事タイトルを日本語に意訳する（英語記事の場合）
3. 要約を1〜2文・40〜60字で書く
4. **news-analyzer スキル**に記載のフォーマットでメッセージを組み立てる

メッセージの日付は日本時間（JST）で表示する。

### 4. メッセージをファイルに書き出す

```bash
# メッセージを /tmp/digest-message.txt に書き込む
# （Write ツールを使って直接書き込む）
```

### 5. LINEに送信する

**⚠️ 必ずこのスクリプトを使用すること。curl で直接送信しないこと。**

```bash
node .claude/skills/line-sender/scripts/send-message.js /tmp/digest-message.txt
```

## 注意事項

- 記事が1件も取得できないカテゴリはそのセクションを省略する
- Cursorカテゴリは記事がなくても必ずセクションを表示し「本日の更新はありませんでした」と記載する
- URLは短縮せずそのまま掲載する
- すべての説明は**読者視点（AIに興味がある人）** で書く
